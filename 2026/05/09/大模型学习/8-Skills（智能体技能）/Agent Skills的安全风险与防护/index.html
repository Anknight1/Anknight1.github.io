第十篇：Agent Skills的安全风险与防护

发表于2026-05-09|更新于2026-04-29|大模型学习8-Skills（智能体技能）
|浏览量:

前言

本文是「大模型学习」系列文章之一，持续更新中。

Agent Skills 是一把双刃剑。它让 AI Agent 具备了强大的可扩展能力，但也引入了新的攻击面。2025 年，Snyk 对 ClawHub（最大的 Agent Skills 市场）进行了首次全面安全审计，结果触目惊心：36% 的技能存在安全漏洞，13.4% 包含严重级别问题，76 个已确认的恶意 payload 在审计时仍处于活跃状态。

本文系统梳理 Agent Skills 的安全威胁模型，并给出防御建议。

正文

一、为什么 Skills 天然容易被攻击

Skills 的核心机制是：将 Markdown 文件中的每一行都作为指令注入到 Agent 的上下文中。这个设计让 Skills 极其强大，但也让提示注入（Prompt Injection）变得异常简单——攻击者只需要在 SKILL.md 中写入恶意指令，Agent 就会忠实地执行。

与传统软件漏洞不同，Skills 的攻击面有几个独特之处：

1. 指令即代码：SKILL.md 中的每一行都是 Agent 的”代码”，没有编译器、没有类型检查、没有沙箱。

2. 信任传递：用户信任某个 Skill，就等于信任了 Skill 作者写的所有指令，包括可能隐藏在文件深处的恶意内容。

3. 供应链风险：从 Skills 市场安装的技能，其内容可能在安装后被悄悄修改（类似 npm 包投毒）。

4. 绕过审批：即使 Agent 有”需要用户确认”的机制，精心构造的提示注入也可能让 Agent 相信某个操作已经被授权。

二、ToxicSkills：Snyk 的安全审计报告

2025 年，Snyk 发布了 ToxicSkills 报告，对 ClawHub 上的 Skills 进行了全面扫描。主要发现：

数据概览：

  • 扫描技能总数：4,200+
  • 存在安全漏洞的比例：36%
  • 包含严重级别问题的比例：13.4%
  • 已确认的恶意 payload 数量：76 个
  • 审计时仍处于活跃状态的恶意技能：8 个

主要攻击类型分布：

攻击类型 占比 描述
提示注入 42% 通过隐藏指令操控 Agent 行为
凭证窃取 28% 诱导 Agent 读取并外泄 API Key、密码等
数据外泄 19% 将用户文件内容发送到攻击者服务器
权限提升 11% 绕过 Agent 的权限检查机制

四、OWASP Agentic Skills Top 10

OWASP 在 2025 年发布了专门针对 Agent Skills 的 Top 10 安全风险：

AS01 - 提示注入（Prompt Injection）

最常见的攻击类型。攻击者在 SKILL.md 中嵌入覆盖系统指令的内容，让 Agent 执行未授权操作。

AS02 - 不安全的工具调用（Insecure Tool Invocation）

技能诱导 Agent 调用危险工具（如 shell 命令执行、文件系统写入）而不经过适当的权限检查。

AS03 - 凭证泄露（Credential Exposure）

技能指令要求 Agent 读取并暴露敏感文件（.env、/.ssh/、/.aws/ 等）。

AS04 - 供应链投毒（Supply Chain Poisoning）

从技能市场安装的技能在安装后被修改，注入恶意内容。

AS05 - 过度权限（Excessive Permissions）

技能请求超出其功能所需的权限范围。

AS06 - 数据外泄（Data Exfiltration）

技能通过网络请求将用户数据发送到外部服务器。

AS07 - 持久化后门（Persistent Backdoor）

技能修改系统配置（如 shell 配置文件、cron 任务），在技能卸载后仍保持恶意行为。

AS08 - 社会工程（Social Engineering）

技能通过伪造的”系统消息”或”管理员指令”欺骗用户授权危险操作。

AS09 - 依赖混淆（Dependency Confusion）

技能引用的外部脚本或资源被替换为恶意版本。

AS10 - 审批绕过（Approval Bypass）

技能通过声称”已预先授权”来绕过 Agent 的确认机制。

总结

Agent Skills 的安全问题本质上是提示注入问题的一个特殊形式——攻击者通过控制 Agent 的”指令来源”（SKILL.md 文件）来操控 Agent 的行为。防御的核心思路是：最小化信任（只信任来源可验证的技能）、最小化权限（技能只能访问它需要的资源）、最大化可见性（记录所有技能调用和工具执行）。

随着 Agent Skills 生态的快速发展，这个领域的安全问题会越来越受到重视。OWASP 的 Agentic Skills Top 10 和 Snyk 的 ToxicSkills 报告都是很好的参考资料。
