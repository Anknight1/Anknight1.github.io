第五篇：一些好用的Skills推荐

发表于2026-05-03|更新于2026-04-29|大模型学习8-Skills（智能体技能）
|浏览量:

前言

本文是「大模型学习」系列文章之一，持续更新中。

本篇整理了社区中经过验证的高质量 Skills，按使用场景分类推荐，帮助你快速找到适合自己的 Skill。

正文

一、代码质量类

Code-Review-Expert

以资深工程师视角进行结构化代码审查，覆盖架构设计、SOLID 原则、安全性、性能问题、错误处理、边界条件等维度。

  • 仓库：github.com/sanyuan0704/code-review-expert
  • 触发场景：代码 Review、PR 审查、代码质量检查

TDD（测试驱动开发）

引导 AI 按照测试驱动开发的流程工作：先编写测试用例，观察它是否失败，然后编写最少的代码使其通过测试。

  • 仓库：github.com/obra/superpowers
  • 触发场景：编写新功能、重构代码

Simplify（Claude Code 内置）

审查最近修改的文件，从复用性、质量、效率三个维度进行检查，并自动修复发现的问题。支持并行多代理审查，适合功能开发或 Bug 修复后的代码清理。

  • 触发方式：/simplify
  • 触发场景：功能开发完成后的代码清理

二、Git 工作流类

Git Commit with Conventional Commits

自动分析 diff，智能暂存文件并生成符合 Conventional Commits 规范的语义化 commit message。

  • 仓库：github.com/github/awesome-copilot
  • 触发场景：提交代码、生成 commit message

Batch（Claude Code 内置）

大规模批量修改代码库，自动将任务拆分，每个任务在隔离的 git worktree 中执行，可批量创建 PR。

  • 触发方式：/batch <指令>
  • 触发场景：全局重命名、批量格式化、大规模重构

三、文档处理类

document-skills（Anthropic 官方插件）

包含 PDF、Word（docx）、Excel（xlsx）、PowerPoint（pptx）四个 Skills，支持文档的创建、编辑、分析和格式转换。

  • 安装：claude plugin install document-skills@anthropic-agent-skills
  • 触发场景：处理各类办公文档

四、调试和排查类

Debug（Claude Code 内置）

排查当前 Claude Code 会话中的问题，读取 debug log 末尾 20 行进行分析。

  • 触发方式：/debug [描述]
  • 触发场景：Claude Code 行为异常、工具调用失败

Stuck（Claude Code 内置）

当 AI 卡住或陷入循环时，提供诊断和建议，帮助找到突破口。

  • 触发方式：/stuck
  • 触发场景：任务执行陷入死循环、AI 反复尝试同一方法失败

五、内容创作类

Content-Research-Writer

结合网络搜索和内容写作能力，先研究主题，再生成高质量的文章或报告。

  • 仓库：skills.deeptoai.com
  • 触发场景：写技术博客、调研报告、产品文档

六、Superpowers 技能集合

Superpowers 是一个知名的 Skills 集合仓库，包含多个经过验证的高质量 Skills：

其中包含的 Skills 有：测试驱动开发（TDD）、代码重构、API 设计、数据库优化等。

七、如何找到更多 Skills

skills.sh：第三方 Skill 目录站，可按关键词搜索，直接给出安装命令。

GitHub 搜索：搜索 SKILL.md 或 agent-skills 关键词，可以找到大量社区贡献的 Skills。

Claude Code 内置：通过 What skills are available? 查看当前已安装的所有 Skills。

八、安装推荐

对于刚开始使用 Skills 的用户，推荐按以下顺序安装：

第一步，安装 Anthropic 官方文档处理插件，覆盖最常见的文档处理需求：

第二步，安装 Superpowers 技能集合，获得代码质量相关的 Skills：

第三步，根据自己的具体需求，在 skills.sh 上搜索并安装专项 Skills。

总结

好的 Skills 能显著提升 AI 的工作质量和一致性。选择 Skills 时，优先考虑：来源可信（官方或知名开源项目）、有持续维护（近期有更新）、描述清晰（触发条件明确）。

安装前务必检查 scripts 目录中的脚本内容，避免安全风险。
