第七篇：OpenCode中Skills的实现原理

发表于2026-05-06|更新于2026-04-29|大模型学习8-Skills（智能体技能）
|浏览量:

前言

本文是「大模型学习」系列文章之一，持续更新中。

OpenCode 是 Claude Code 最有力的开源挑战者，2025 年 6 月上线，半年内 GitHub Star 突破 10 万。它不仅兼容 Claude Code 的 SKILL.md 格式，还在 Skills 系统上做了大量扩展——更多的发现路径、更严格的权限控制、更完善的插件生态。

本文从源码角度拆解 OpenCode 的 Skills 实现，重点关注它与 Claude Code 的异同。

正文

一、OpenCode Skills 概览

OpenCode 的 Skills 系统与 Claude Code 高度兼容，核心设计理念相同：通过 SKILL.md 文件定义可复用的 Agent 指令，在会话开始时注入技能列表，Agent 按需加载完整内容。

与 Claude Code 的主要差异：

对比维度 Claude Code OpenCode
发现路径 2 个（项目 + 全局） 6 个（含插件缓存、市场）
权限控制 无内置权限系统 细粒度权限（allow/deny/ask）
插件生态 无 支持插件市场
格式兼容 原生 完全兼容 Claude Code 格式
工具名称 read_skill skill
注入格式 <available_skills> XML <available_skills> XML（相同）

四、YAML Frontmatter 验证

OpenCode 使用 Zod 对 SKILL.md 的 frontmatter 进行严格验证：

name 字段的规则：

  • 1-64 个字符
  • 只允许小写字母、数字、单个连字符
  • 不能以 - 开头或结尾
  • 不能有连续的 --
  • 必须与所在目录名一致

description 字段： 1-1024 个字符，是 Agent 判断是否使用该技能的唯一依据，要写得足够具体。

八、SKILL.md 格式示例

frontmatter 字段说明：

  • name（必需）：技能唯一标识，必须与目录名一致
  • description（必需）：技能描述，Agent 据此判断是否使用
  • license（可选）：许可证类型
  • compatibility（可选）：兼容性标记（opencode/claude/all）
  • metadata（可选）：自定义键值对，可用于分类和过滤

九、与 Claude Code 的兼容性

OpenCode 完全兼容 Claude Code 的 SKILL.md 格式，可以直接使用 Claude Code 的技能：

OpenCode 还支持 Claude Code 的插件缓存目录（~/.claude/plugins/cache/），可以直接使用通过 Claude Code 安装的插件技能。

唯一的差异是工具名称：Claude Code 使用 read_skill，OpenCode 使用 skill。如果你的 SKILL.md 中有引用工具名称的说明，需要注意这个区别。

总结

OpenCode 的 Skills 实现在 Claude Code 的基础上做了大量扩展：6 个发现路径（含插件生态）、细粒度权限控制、Zod 严格验证、可执行脚本自动发现。核心设计理念与 Claude Code 完全一致——渐进式披露，先注入元数据，按需加载完整内容。

对于已经在使用 Claude Code Skills 的开发者，迁移到 OpenCode 几乎零成本，现有的 SKILL.md 文件可以直接复用。
