第六篇：Spring AI中Skill的使用方法

前言

本文是「大模型学习」系列文章之一，持续更新中。

前几篇介绍了 Agent Skills 的概念和在 Claude Code 中的使用。本篇介绍如何在 Spring AI Alibaba 中集成和使用 Skills，覆盖两种主要方式：ReactAgent + SkillsAgentHook（适合复杂 Agent 场景）和 ChatClient + SkillPromptAugmentAdvisor（适合简单对话增强场景）。

两种集成方式的对比

Spring AI Alibaba 提供了两种 Skills 集成方式：

对比维度 ReactAgent + SkillsAgentHook ChatClient + SkillPromptAugmentAdvisor
适用场景 需要 Agent 自主决策的复杂任务 简单的对话增强，不需要 Agent 循环
read_skill 工具 自动注册，模型可按需加载完整 SKILL.md 不自动注册，需要手动配置
技能发现 完整的渐进式披露（元数据 + 按需加载） 只有技能发现（元数据注入），无自动加载
复杂度 较高 较低
适合场景 需要执行脚本、调用工具的复杂技能 只需要注入知识和指令的简单技能

选型建议：

如果你的 Skill 需要执行 Python 脚本、调用 Shell 命令或使用其他工具，选择 ReactAgent + SkillsAgentHook。如果你的 Skill 只是提供知识和指令（不需要执行代码），选择 ChatClient + SkillPromptAugmentAdvisor，更简单轻量。

方式一：ReactAgent + SkillsAgentHook

Spring AI Alibaba 中的 Skills 架构

工作原理：

SkillsAgentHook 在每次 Agent 执行前，将所有可用技能的元数据（name、description、skillPath）注入到系统提示中，同时注册一个 read_skill 工具。当模型判断需要某个技能时，会调用 read_skill(skill_name) 加载完整的 SKILL.md 内容，然后按照技能指令执行任务。

这正是渐进式披露的 Java 实现：元数据常驻系统提示，完整内容按需加载。

最佳实践

控制 SKILL.md 大小：单文件建议约 1500-2000 tokens，长内容放 references/ 目录并在正文中列出路径。

技能名称保持一致：SKILL.md 中的 name 字段、read_skill 调用参数、groupedTools 的 key 必须完全一致。

按需使用 groupedTools：只有需要”随技能激活”的工具才用 groupedTools，其他工具直接用 Agent 的 .tools() 方法注册。

项目级覆盖用户级：同名技能时，项目级（projectSkillsDirectory）会覆盖用户级（userSkillsDirectory），利用这个特性可以在项目中定制化覆盖通用技能。

控制技能加载时机：lazyLoad(true) 首次请求时才加载技能，适合技能数量多的场景；lazyLoad(false) 启动时加载，适合技能数量少、需要快速响应的场景。

总结

Spring AI Alibaba 提供了两种 Skills 集成方式，各有适用场景：

ReactAgent + SkillsAgentHook 通过 SkillRegistry + SkillsAgentHook 的组合，为 Java 开发者提供了完整的 Agent Skills 支持。核心流程是：创建 SkillRegistry → 创建 SkillsAgentHook（注入技能列表 + 注册 read_skill 工具）→ 将 Hook 挂载到 ReactAgent。渐进式披露在 Java 端的实现方式是：系统提示中只注入技能元数据，模型通过 read_skill 工具按需加载完整 SKILL.md，配合 groupedTools 实现工具的渐进式激活。

ChatClient + SkillPromptAugmentAdvisor 是一种轻量级的 Skills 集成方式，适合不需要执行代码的简单技能场景。它通过 Advisor 机制在每次请求前将技能元数据注入系统提示，让模型知道有哪些技能可用。如果需要完整的渐进式披露（包括按需加载完整 SKILL.md），需要额外配置 read_skill 工具，或者改用 ReactAgent 方案。
