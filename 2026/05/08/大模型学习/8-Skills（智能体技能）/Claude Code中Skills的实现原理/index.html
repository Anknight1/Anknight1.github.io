正文

一、Skills 系统的整体架构

Claude Code 的 Skills 系统位于源码的 skills/ 目录下，共约 20 个文件，实现了一个三来源技能系统：

核心文件：

文件 职责

loadSkillsDir.ts 文件系统技能加载器，从磁盘扫描用户自定义 Skills
mcpSkillBuilders.ts MCP 技能构建器，解决循环依赖问题

whenToUse 字段是自动触发的核心依据，Claude 会根据这个字段判断当前任务是否应该激活该 Skill。

getPromptForCommand 是 Skill 的核心方法，返回注入到模型对话中的 Prompt 内容。

关键细节：脚本执行

当 Skill 的指令中引用了可执行脚本时，Claude Code 会运行脚本，但只将脚本的输出结果注入上下文，脚本代码本身不会进入上下文窗口。这是一个重要的设计——既保证了确定性（脚本执行结果是确定的），又节省了 Token（不需要把脚本代码也塞进去）。

七、安全机制

文件写入安全：

当技能包含 files 字段时，Claude Code 使用特殊的文件权限写入：

O_NOFOLLOW 防止符号链接攻击（不跟随符号链接），O_EXCL 防止文件已存在时覆盖，0o600 确保文件只有所有者可读写。

工具白名单：

通过 allowedTools 字段，每个 Skill 只能使用明确授权的工具，防止 Skill 越权操作：

独立上下文（fork）：

设置 context: fork 的 Skill 在独立的上下文中执行，不会访问或污染主对话的上下文，适合需要隔离执行的敏感操作。

8-Skills 与 CLAUDE.md 的区别

很多人会混淆 Skills 和 CLAUDE.md，它们的定位完全不同：

对比维度 CLAUDE.md Skills
加载时机 始终加载（每次对话都注入） 按需加载（只在触发时注入）
内容类型 项目级别的全局规范和背景 特定任务的执行 SOP
Token 消耗 每次对话都消耗 只在触发时消耗
适用场景 项目技术栈、代码风格、架构说明 代码 Review、部署检查、问题排查
触发方式 无需触发，自动注入 自动匹配或手动调用

最佳实践：把”每次对话都需要知道的信息”放在 CLAUDE.md，把”特定任务才需要的 SOP”封装成 Skill。

总结

Claude Code 的 Skills 系统通过三来源架构（内置 + 文件系统 + MCP）实现了灵活的技能扩展机制。

从实现原理来看，Skills 本质上是带 Frontmatter 的 Markdown Prompt——轻量但强大。当 Skill 被触发时，getPromptForCommand 方法读取 SKILL.md 的正文内容，将其作为 Prompt 注入到模型对话中，模型再根据 allowedTools 白名单使用相应的工具执行任务。

安全方面，Claude Code 通过文件写入权限控制（O_NOFOLLOW + 0o600）、工具白名单（allowedTools）和独立执行上下文（context: fork）三重机制，确保 Skills 在安全的边界内运行。
