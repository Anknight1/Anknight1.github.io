第十一篇：Hermes Agent的Skill自进化机制

发表于2026-05-10|更新于2026-04-29|大模型学习8-Skills（智能体技能）
|浏览量:

前言

本文是「大模型学习」系列文章之一，持续更新中。

前面介绍的 Claude Code、OpenCode、Spring AI Alibaba 中的 Skills，都有一个共同点：技能是人工编写的。你写多少，Agent 会多少；你不写，它就不会。

Hermes Agent 做了一件这些框架架构上做不了的事：Agent 干完活之后，会自动把踩坑经验提炼成可复用的 Skill，下次遇到同类问题直接调用。用得越久，能力越强。

这不是功能差异，是设计哲学的分野——一个靠人喂，一个自己长。

正文

一、Hermes Agent 简介

Hermes Agent 是 Nous Research 于 2026 年 2 月开源的自主 AI Agent 框架。上线不到半年，GitHub Star 突破 10 万，在 OpenRouter 排行榜上 Top Coding Agents 排第一、Top Productivity 排第二，增速 +204%。

它的核心定位是：The agent that grows with you（与你共同成长的 Agent）。

打个比方：Memory 是助理随身带的小本子，记着”老板喜欢喝美式”这些事实；Skill 是助理积累的操作手册——“部署 K8s 第 2 步一定要先推镜像”；Nudge Engine 是定时响的闹钟，提醒助理回头想想有没有什么值得记的。

四、Skill 的自动改进机制

Hermes 不只是创建 Skill，还会在每次使用 Skill 后自动改进它：

改进的触发条件：

  • 实际执行步骤与 Skill 描述有偏差（说明 Skill 描述不够准确）
  • 执行过程中发现了新的注意事项
  • 遇到了 Skill 中未提及的错误及其解决方案

五、Nudge Engine：推动 Agent 主动沉淀

Nudge Engine 是 Hermes 自进化机制的”催化剂”。它在会话结束时或特定时间点触发，提醒 Agent 回顾本次会话，判断是否有值得沉淀的经验：

七、与 Claude Code Skills 的对比

对比维度 Claude Code Skills Hermes Agent Skills
创建方式 人工编写 SKILL.md 自动从经验提炼 + 人工编写
更新方式 人工修改 每次使用后自动改进
存储格式 Markdown 文件 JSON 文件（含统计信息）
发现机制 文件系统扫描 文件系统扫描 + 语义搜索
版本管理 无（依赖 git） 内置版本号
使用统计 无 使用次数、成功率
跨会话学习 无 有（Nudge Engine）

八、自进化的边界

Hermes 的自进化能力令人印象深刻，但也有其局限性：

能自动学习的： 操作步骤、工具配置、错误处理方案、环境特定的注意事项。

不能自动学习的： 需要外部知识的技能（如特定领域的专业知识）、需要人工判断的业务逻辑、涉及安全敏感操作的流程（这类应该由人工审核）。

潜在风险： 如果 Agent 在某次任务中采用了错误的方法并”成功”完成了任务（但实际上留下了隐患），这个错误方法可能被提炼为 Skill，在未来被重复使用。因此，Hermes 建议定期审查自动生成的 Skill，删除或修正不准确的内容。

总结

Hermes Agent 的 Skill 自进化机制代表了 Agent Skills 发展的一个重要方向：从”人工编写、静态存储”走向”自动提炼、动态演化”。通过 Nudge Engine 的定时推动、任务完成后的自动提炼、每次使用后的自动改进，Hermes 实现了真正意义上的”越用越强”。

这种设计哲学的核心洞察是：Agent 的能力不应该只取决于开发者预先写了多少 Skill，而应该随着使用积累而不断增长。这对于个人助理类 Agent 尤其有价值——每个用户的使用习惯和工作环境都不同，自动学习的 Skill 比通用的预设 Skill 更贴合个人需求。
