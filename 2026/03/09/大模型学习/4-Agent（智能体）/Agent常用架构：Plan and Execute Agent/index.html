前言

本文是「大模型学习」系列文章之一，持续更新中。

ReAct Agent 在处理简单任务时表现良好，但面对复杂的多步骤任务时，容易陷入局部最优，缺乏全局规划。Plan and Execute Agent 通过”先规划、再执行”的方式解决这一问题。

Plan and Execute vs ReAct

对比维度 ReAct Plan and Execute
规划方式 逐步规划 全局规划
适用任务 简单、短任务 复杂、多步骤任务
LLM 调用次数 较少 较多（规划+执行）
灵活性 高（随时调整） 中（按计划执行）
可解释性 低 高（计划清晰）

适用场景

Plan and Execute 特别适合：

  • 研究任务：需要多步骤收集、分析、总结信息
  • 代码生成：先设计架构，再逐步实现
  • 数据分析：先制定分析方案，再执行各步骤
  • 内容创作：先规划大纲，再逐章撰写

总结

Plan and Execute Agent 通过”先规划、再执行”的方式，解决了 ReAct 在复杂任务上缺乏全局视野的问题。其核心是将任务分解为 Planner、Executor、Replanner 三个角色，各司其职，协同完成复杂任务。
