第三篇：Agent常用架构：ReAct Agent

发表于2026-03-03|更新于2026-04-29|大模型学习4-Agent（智能体）
|浏览量:

前言

本文是「大模型学习」系列文章之一，持续更新中。

ReAct（Reasoning + Acting）是目前最主流的 Agent 架构，由 Google 在 2022 年提出。它将 LLM 的推理能力与工具调用结合，让 Agent 能够像人类一样”边思考边行动”。

ReAct 的核心思想

ReAct 的名字来自 Reasoning（推理）+ Acting（行动）的组合。

传统方法的问题：

  • 纯推理（Chain-of-Thought）：只思考，不行动，无法获取外部信息
  • 纯行动（Action-only）：只执行，不思考，容易出错

ReAct 的创新：将推理和行动交织在一起，每次行动前先思考，行动后再观察结果，形成闭环。

方式二：基于 Function Calling（现代方式）

利用 LLM 的 Function Calling 能力，直接返回结构化的工具调用：

优点： 结构化输出，解析可靠，是目前的主流方式。

ReAct 的局限性

串行执行： 每次只能执行一个工具，效率较低。

无法并行： 即使两个工具调用相互独立，也只能串行执行。这是 ReWOO 和 LLMCompiler 要解决的问题。

上下文限制： 随着步骤增多，上下文越来越长，可能超出 LLM 的 Context Window。

ReAct vs 其他架构

架构 特点 适用场景
ReAct 串行推理+行动 通用场景，简单任务
Plan-and-Execute 先规划再执行 复杂多步骤任务
ReWOO 并行工具调用 需要多工具并行的场景
Reflection 自我反思改进 需要高质量输出的场景

总结

ReAct 是 Agent 领域最基础、最重要的架构，其核心思想是将推理和行动交织在一起，通过”思考-行动-观察”的循环逐步完成任务。

理解 ReAct 是理解所有其他 Agent 架构的基础，后续的 Plan-and-Execute、ReWOO 等架构都是在 ReAct 的基础上针对特定问题的改进。
