前言

本文是「大模型学习」系列文章之一，持续更新中。

2025 年 4 月，Google 发布了 A2A（Agent-to-Agent）协议，旨在标准化不同 Agent 框架之间的通信方式。A2A 与 MCP 互补，共同构建了 AI Agent 的互联互通生态。

A2A 协议简介

A2A 解决的问题：

在 Multi Agent 系统中，不同的 Agent 可能使用不同的框架（LangGraph、AutoGen、Spring AI Alibaba 等）。没有标准协议时，Agent 之间的通信需要定制化开发，难以互操作。

A2A 提供了一套标准化的 HTTP 协议，让任何遵循 A2A 规范的 Agent 都能相互通信。

A2A 与 MCP 的协同

总结

A2A 协议通过标准化 Agent 间通信，构建了开放的 Agent 互联互通生态：

  • Agent Card：标准化的能力描述，支持自动发现
  • Task 模型：统一的任务提交和状态管理
  • 流式支持：支持 SSE 流式输出
  • 与 MCP 互补：MCP 解决工具调用，A2A 解决 Agent 协作

随着 A2A 生态的成熟，不同厂商的 Agent 将能够无缝协作。
