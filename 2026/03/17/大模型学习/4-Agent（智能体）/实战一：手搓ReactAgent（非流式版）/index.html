总结

通过手搓 ReAct Agent，我们深入理解了其底层原理：

非流式版本的核心是消息历史维护和工具调用循环：LLM 返回工具调用 → 执行工具 → 结果加入历史 → 再次调用 LLM，直到 LLM 不再调用工具。

流式版本的关键挑战是增量工具调用解析：工具调用的 JSON 是分块传输的，需要累积后再解析；同时通过事件模型（TextToken、ToolCallStart、ToolCallResult、AgentComplete）让前端实时感知 Agent 的执行状态。
