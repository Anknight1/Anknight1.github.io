第十篇：Agent常用架构：ReWOO与LLMCompiler

发表于2026-03-10|更新于2026-04-29|大模型学习4-Agent（智能体）
|浏览量:

前言

本文是「大模型学习」系列文章之一，持续更新中。

ReAct 和 Plan-and-Execute 都是串行执行工具调用，当任务需要多个独立工具时效率较低。ReWOO 和 LLMCompiler 通过并行化工具调用，显著提升了 Agent 的执行效率。

ReWOO：无观察推理

ReWOO（Reasoning WithOut Observation）的核心思想：先一次性规划所有工具调用，再并行执行，最后汇总结果。

ReWOO vs ReAct

ReWOO 的三个阶段

阶段一：Planner（规划器）

阶段二：Worker（并行执行）

阶段三：Solver（求解器）

LLMCompiler：编译器模式

LLMCompiler 将 Agent 的执行过程类比为编译器：将任务”编译”为可并行执行的 DAG（有向无环图）。

ReWOO vs LLMCompiler vs ReAct

对比维度 ReAct ReWOO LLMCompiler
执行方式 串行 并行（无依赖） 并行（DAG 调度）
规划次数 每步规划 一次规划 一次规划
依赖处理 自然串行 简单引用 DAG 依赖图
适用场景 简单任务 独立工具并行 复杂依赖任务
实现复杂度 低 中 高

性能对比

总结

ReWOO 和 LLMCompiler 通过并行化工具调用，显著提升了 Agent 的执行效率：

  • ReWOO：一次性规划所有工具调用，并行执行，适合工具间相互独立的场景
  • LLMCompiler：将任务组织为 DAG，按依赖关系并行调度，适合复杂依赖场景

在工具调用耗时较长（如网络请求、数据库查询）的场景下，并行化可以带来显著的性能提升。
