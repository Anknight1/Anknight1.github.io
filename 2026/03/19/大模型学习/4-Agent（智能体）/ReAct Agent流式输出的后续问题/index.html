第十八篇：ReAct Agent流式输出的后续问题

发表于2026-03-19|更新于2026-04-29|大模型学习4-Agent（智能体）
|浏览量:

前言

本文是「大模型学习」系列文章之一，持续更新中。

上一篇实现了流式 ReAct Agent，但在实际使用中还会遇到一些棘手的问题。本文深入探讨这些后续问题及其解决方案。

问题一：工具调用期间的”空白期”

问题描述：

当 Agent 调用工具时（如搜索、数据库查询），LLM 暂停输出，用户看到的是一段”空白”，不知道 Agent 在做什么。

解决方案：发送工具执行状态事件

前端展示：

问题二：工具调用 JSON 的增量解析

问题描述：

OpenAI 的流式 API 中，工具调用的 JSON 是分块传输的：

需要累积所有 chunk 才能得到完整的工具调用。

解决方案：ToolCallAccumulator

问题三：多个工具并发调用时的流式处理

问题描述：

当 LLM 一次返回多个工具调用时（parallel tool calls），如何处理流式输出？

解决方案：并行执行 + 合并结果

问题四：流式输出中断后的恢复

问题描述：

网络中断或客户端断开连接时，Agent 的执行状态如何保存和恢复？

解决方案：Checkpointer + 断点续传

问题五：流式输出的 Token 计数

问题描述：

流式输出时，如何统计 Token 使用量（用于计费或限流）？

解决方案：在流结束时获取 usage 信息

问题六：工具执行超时处理

最佳实践总结

问题 解决方案
工具执行空白期 发送 tool_start/tool_complete 事件
工具调用 JSON 增量 ToolCallAccumulator 累积
多工具并发 CompletableFuture 并行执行
连接中断恢复 Checkpointer + threadId
Token 计数 stream_options.include_usage
工具超时 CompletableFuture.get(timeout)

总结

流式 ReAct Agent 在实际使用中会遇到各种边界情况，需要仔细处理：

  • 工具执行期间的用户体验（加载动画、进度提示）
  • 工具调用 JSON 的增量解析
  • 并发工具调用的处理
  • 断线重连和状态恢复

这些问题的解决方案共同构成了生产级流式 Agent 的基础。
