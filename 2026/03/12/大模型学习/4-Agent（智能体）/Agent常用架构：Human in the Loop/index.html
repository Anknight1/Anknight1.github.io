第十二篇：Agent常用架构：Human in the Loop

发表于2026-03-12|更新于2026-04-29|大模型学习4-Agent（智能体）
|浏览量:

前言

本文是「大模型学习」系列文章之一，持续更新中。

完全自主的 Agent 在某些场景下存在风险：执行不可逆操作（删除文件、发送邮件）、处理敏感数据、做出重要决策等。Human in the Loop（HITL）架构在关键节点引入人工审核，平衡了 Agent 的自主性与安全性。

Human in the Loop 的核心思想

三种介入模式

模式一：审批模式（Approve/Reject）

适用场景

Human in the Loop 特别适合以下场景：

高风险操作：

  • 删除重要文件或数据
  • 发送邮件或消息
  • 执行数据库写操作
  • 调用付费 API

合规要求：

  • 金融交易审批
  • 医疗决策辅助
  • 法律文件生成

质量把控：

  • 内容发布前审核
  • 代码部署前确认

HITL 的设计原则

  1. 最小化干预：只在真正必要的节点引入人工审核，避免过度打扰
  2. 清晰的审核界面：向审核人员清晰展示 Agent 的意图和影响
  3. 超时处理：设置审核超时，避免 Agent 无限等待
  4. 审核日志：记录所有审核决策，便于追溯

总结

Human in the Loop 架构通过在关键节点引入人工审核，在 Agent 自主性和安全性之间取得平衡：

  • 审批模式：高风险操作前请求批准
  • 编辑模式：允许人工修改 Agent 的计划
  • 监控模式：实时监控，随时可以中断

随着 Agent 能力的增强，HITL 将成为生产级 Agent 系统的标配。
