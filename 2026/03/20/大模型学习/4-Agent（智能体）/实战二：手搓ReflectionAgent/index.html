前言

本文是「大模型学习」系列文章之一，持续更新中。

本文手搓一个 Reflection Agent，以”代码生成”为场景，实现生成 → 反思 → 改进的迭代循环，直到代码质量达标为止。

场景设计

我们要实现一个”代码生成助手”：

  1. Generator：根据需求生成 Java 代码
  2. Reflector：评估代码质量，给出改进建议
  3. 循环：直到代码质量评分 ≥ 8 分或达到最大迭代次数

完整实现

数据模型

Generator（代码生成器）

Reflector（代码反思器）

    请从以下维度评估（每项 0-10 分）：
    1. 正确性：代码逻辑是否正确，能否满足需求
    2. 健壮性：是否有适当的错误处理和边界条件处理
    3. 可读性：代码是否清晰易懂，命名是否规范
    4. 性能：是否有明显的性能问题
    5. 测试覆盖：测试用例是否充分
    
    以 JSON 格式返回：
    {
      "scores": {
        "correctness": 8,
        "robustness": 7,
        "readability": 9,
        "performance": 8,
        "testCoverage": 6
      },
      "overallScore": 7.6,
      "isSatisfactory": false,
      "issues": ["问题1", "问题2"],
      "suggestions": ["建议1", "建议2"],
      "summary": "总体评价"
    }
    """;

public ReflectionResult reflect(String requirement, CodeGenerationResult code) {
    String prompt = REFLECT_PROMPT
        .replace("{requirement}", requirement)
        .replace("{code}", code.getCode());

    String response = chatModel.call(prompt);
    return parseReflectionResult(response);
}

private ReflectionResult parseReflectionResult(String json) {
    String jsonStr = extractJson(json);
    JsonObject obj = JsonParser.parseString(jsonStr).getAsJsonObject();

    return ReflectionResult.builder()
        .score(obj.get("overallScore").getAsDouble())
        .isSatisfactory(obj.get("isSatisfactory").getAsBoolean())
        .issues(obj.getAsJsonArray("issues").asList().stream()
            .map(JsonElement::getAsString)
            .collect(Collectors.toList()))
        .suggestions(obj.getAsJsonArray("suggestions").asList().stream()
            .map(JsonElement::getAsString)
            .collect(Collectors.toList()))
        .summary(obj.get("summary").getAsString())
        .build();
}

总结

Reflection Agent 的核心是”生成 → 反思 → 改进”的循环：

  1. Generator：根据需求（和上一轮的反馈）生成内容
  2. Reflector：评估内容质量，给出具体的问题和改进建议
  3. 循环控制：评分达标或达到最大迭代次数时停止

这种模式特别适合代码生成、文章写作等对质量要求较高的场景，通过迭代改进显著提升输出质量。
