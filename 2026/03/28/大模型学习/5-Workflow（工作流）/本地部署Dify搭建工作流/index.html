前言

本文是「大模型学习」系列文章之一，持续更新中。

Dify 是目前最流行的开源 LLM 应用开发平台，支持本地私有化部署。本篇手把手带你完成 Dify 的本地部署，并搭建第一个 AI 工作流。

环境要求

  • Docker 20.10+
  • Docker Compose 2.0+
  • 内存：至少 4GB（推荐 8GB）
  • 磁盘：至少 10GB 可用空间

第三步：启动服务

启动后会拉起以下服务：

  • nginx：反向代理，监听 80 端口
  • web：前端界面
  • api：后端 API 服务
  • worker：异步任务处理
  • db：PostgreSQL 数据库
  • redis：缓存和消息队列
  • weaviate：向量数据库

第四步：初始化管理员账号

浏览器访问 http://localhost，首次访问会跳转到初始化页面，设置管理员邮箱和密码。

第五步：配置模型

进入「设置 → 模型供应商」，添加你的 LLM API Key：

  • OpenAI：填入 OPENAI_API_KEY
  • 通义千问：填入阿里云 API Key
  • DeepSeek：填入 DeepSeek API Key
  • Ollama：填入本地 Ollama 地址（http://host.docker.internal:11434）

常用节点介绍

LLM 节点：调用大语言模型，是工作流的核心节点，支持配置系统提示词、用户提示词、模型参数等。

知识检索节点：从知识库中检索相关内容，支持语义检索和关键词检索，常与 LLM 节点配合实现 RAG。

代码节点：执行 Python 或 JavaScript 代码，用于数据处理、格式转换等操作。

条件分支节点：根据变量值选择不同的执行路径，实现 if-else 逻辑。

迭代节点：对列表数据进行循环处理，类似 for 循环。

HTTP 请求节点：调用外部 API，获取或发送数据。

变量聚合节点：将多个分支的输出合并为一个变量。

常见问题

端口冲突：如果 80 端口被占用，修改 .env 中的 EXPOSE_NGINX_PORT=8080。

模型调用失败：检查 API Key 是否正确，以及网络是否能访问对应的 API 端点。

内存不足：减少并发 Worker 数量，修改 .env 中的 SERVER_WORKER_AMOUNT=1。

总结

Dify 的本地部署非常简单，只需几条命令就能启动完整的 AI 应用开发平台。通过可视化的工作流编排，即使没有编程经验也能快速构建复杂的 AI 应用。
