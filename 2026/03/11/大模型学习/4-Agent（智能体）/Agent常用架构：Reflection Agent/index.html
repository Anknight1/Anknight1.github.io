第十一篇：Agent常用架构：Reflection Agent

前言

本文是「大模型学习」系列文章之一，持续更新中。

Reflection Agent 引入了”自我反思”机制，让 Agent 能够评估自己的输出质量，并通过迭代改进来生成更高质量的结果。这种架构特别适合需要高质量输出的场景，如代码生成、文章写作、方案设计等。

Reflection 的核心思想

人类在完成复杂任务时，通常会经历”写作 → 审查 → 修改”的循环。Reflection Agent 模拟了这一过程：

基本 Reflection 架构

Generator（生成器）

Reflector（反思器）

Reflection Agent 主循环

高级 Reflection：外部反馈

除了自我反思，还可以引入外部反馈（如代码执行结果、测试结果）：

Reflection vs 其他架构

对比维度 ReAct Plan-Execute Reflection
核心机制 推理+行动 规划+执行 生成+反思
适用场景 信息检索 复杂任务 高质量输出
迭代方式 工具调用循环 步骤执行 质量改进循环
输出质量 中 中高 高
耗时 中 高 高

总结

Reflection Agent 通过”生成 → 反思 → 改进”的循环，显著提升了输出质量：

  • 自我反思：LLM 评估自己的输出，给出改进建议
  • 外部反馈：代码执行、测试结果等客观反馈
  • Reflexion：引入长期记忆，积累跨任务的经验

适合代码生成、文章写作、方案设计等对质量要求较高的场景。
