第十六篇：Agent、AI Workflow与Agentic Workflow的区别

发表于2026-04-09|更新于2026-04-29|大模型学习5-Workflow（工作流）
|浏览量:

前言

本文是「大模型学习」系列文章之一，持续更新中。

在 AI 应用开发中，Agent、AI Workflow、Agentic Workflow 这三个概念经常被混用，让人困惑。本篇用最直白的方式厘清三者的本质区别。

一句话区分

AI Workflow：我提前画好了地图，按图索骥。

Agent：我只知道目的地，自己找路。

Agentic Workflow：整体路线我规划好了，但某些路段让 AI 自己决定怎么走。

详细解释

AI Workflow（AI 工作流）

AI Workflow 是预定义的自动化流程，开发者在设计阶段就确定了所有步骤和分支。

Agent（智能体）

Agent 是具有自主决策能力的 AI 系统，它根据目标和当前状态，自主决定下一步行动。

Agentic Workflow（智能工作流）

Agentic Workflow 是在工作流框架中嵌入 Agent 能力的混合架构。

三者的关系

Agentic Workflow 是 AI Workflow 和 Agent 的融合，在两个极端之间取得平衡。

实际案例对比

场景：客服系统

方案 实现方式 优缺点
AI Workflow 意图分类 → 知识库检索 → 生成回复（固定流程） 可控、稳定，但无法处理复杂问题
Agent LLM 自主决定如何回答，可调用多种工具 灵活，但成本高、不可预测
Agentic Workflow 固定流程处理常见问题，复杂问题交给 Agent 兼顾效率和灵活性

场景：代码审查

方案 实现方式
AI Workflow 静态分析 → LLM 审查 → 生成报告（固定步骤）
Agent LLM 自主决定：读代码 → 运行测试 → 分析错误 → 提建议（循环）
Agentic Workflow 固定流程：提交代码 → [Agent 深度审查] → 生成报告

选择指南

在 Dify 中的对应关系

概念 Dify 中的实现
AI Workflow Dify 工作流（Workflow 应用）
Agent Dify Agent 应用
Agentic Workflow 工作流中嵌入 Agent 节点

总结

三个概念的核心区别在于决策主体和执行路径的确定性：

AI Workflow 的决策主体是开发者（设计时），执行路径完全确定；Agent 的决策主体是 LLM（运行时），执行路径动态生成；Agentic Workflow 是两者的结合，整体框架由开发者确定，局部决策由 LLM 自主完成。

理解这三个概念的本质区别，有助于在实际项目中选择合适的架构，构建既强大又可靠的 AI 应用。
