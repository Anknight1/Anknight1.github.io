第一篇：什么是长期记忆

发表于2026-04-26|更新于2026-04-29|大模型学习7-长期记忆
|浏览量:

前言

本文是「大模型学习」系列文章之一，持续更新中。

你有没有遇到过这样的场景：和 AI 聊了半天，告诉它你的偏好、你的项目背景，结果第二天重新打开对话，它又变成了一个”陌生人”，什么都不记得了？

这就是 AI Agent 缺乏长期记忆的典型表现。本文将系统介绍 AI Agent 的记忆体系，重点讲解长期记忆的概念、类型和实现思路。

正文

一、AI Agent 的记忆体系

在认知科学中，人类的记忆分为短期记忆（工作记忆）和长期记忆。AI Agent 的记忆体系与此类似，通常分为以下几个层次：

二、短期记忆 vs 长期记忆

维度 短期记忆 长期记忆
生命周期 单次会话，关闭即消失 跨会话持久保存
存储位置 LLM 上下文窗口 外部数据库（向量库、图数据库、关系型数据库）
容量 受 Token 上限约束（通常 128K~200K tokens） 理论上无限
访问方式 直接读取（在上下文中） 需要检索后注入上下文
更新方式 自动追加 需要主动写入/更新
典型用途 当前对话的连贯性 用户偏好、历史经验、知识积累

一个直观的类比：

短期记忆 = 桌面上摊开的文件（随时可见，但桌面有限）
长期记忆 = 文件柜里的档案（需要去找，但容量无限）

三、为什么需要长期记忆？

场景一：个性化服务

有长期记忆的 AI：

场景二：知识积累

AI Agent 在执行任务过程中会积累大量经验：哪些方案有效、哪些路径会失败、用户的特殊需求是什么。如果这些经验不能持久化，每次任务都要从零开始，效率极低。

场景三：跨会话任务

复杂任务往往需要多次会话才能完成。没有长期记忆，Agent 无法在下次会话中继续上次的工作进度。

四、长期记忆的四种类型详解

1. 情节记忆（Episodic Memory）

情节记忆记录的是具体的事件和经历，带有时间戳和上下文信息。

情节记忆的特点：

  • 具有时间顺序
  • 包含上下文信息
  • 可以用于推断用户的当前状态和需求

2. 语义记忆（Semantic Memory）

语义记忆存储的是通用知识和事实，不依赖于特定时间或事件。

语义记忆通常通过向量数据库存储，支持语义相似度检索。

3. 程序性记忆（Procedural Memory）

程序性记忆记录的是如何完成某类任务的步骤和经验，类似于”操作手册”。

Manus 团队在上下文工程中提到的”将文件系统用作 Context”，本质上就是一种程序性记忆的实现。

4. 用户偏好记忆（Preference Memory）

用户偏好记忆存储的是用户的个性化偏好和习惯，是实现个性化服务的基础。

五、长期记忆的生命周期

长期记忆的管理涉及四个核心操作：

写入策略：什么信息值得记忆？

不是所有信息都值得写入长期记忆。好的写入策略应该关注：

  • 用户明确表达的偏好（”我喜欢…”、”我不喜欢…”）
  • 重要的事实信息（用户的技术栈、项目背景）
  • 任务执行的关键经验（成功的方案、失败的教训）
  • 用户的个人信息（姓名、职业、所在地）

检索策略：如何找到相关记忆？

六、长期记忆的主流实现方案

方案 特点 适用场景
Mem0 开源，LLM 驱动的智能记忆提取，支持向量+图数据库 生产级 Agent，需要智能记忆管理
LangGraph Memory 与 LangGraph 深度集成，支持多种存储后端 使用 LangGraph 构建的 Agent
Zep 专注于对话记忆，支持知识图谱 对话型 Agent，需要关系推理
Letta（MemGPT） 分层记忆架构，模拟操作系统内存管理 需要复杂记忆管理的长期任务
自定义实现 灵活，可完全控制 特殊需求，现有方案不满足

下一篇，我们将重点介绍目前最流行的长期记忆实现方案——Mem0。

总结

AI Agent 的记忆体系分为短期记忆（Context Window）和长期记忆（外部存储）两大类。长期记忆进一步细分为情节记忆、语义记忆、程序性记忆和用户偏好记忆四种类型。

长期记忆的核心价值在于：让 AI Agent 能够跨会话积累经验、记住用户偏好、持续学习进化，从”金鱼脑”变成真正懂用户的智能伙伴。

实现长期记忆需要解决四个核心问题：写入（什么信息值得记忆）、检索（如何找到相关记忆）、更新（如何处理信息变化）、遗忘（如何清理过时信息）。
