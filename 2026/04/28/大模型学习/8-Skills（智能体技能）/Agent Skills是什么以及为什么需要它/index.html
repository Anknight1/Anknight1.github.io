第一篇：Agent Skills是什么以及为什么需要它

发表于2026-04-28|更新于2026-04-29|大模型学习8-Skills（智能体技能）
|浏览量:

前言

本文是「大模型学习」系列文章之一，持续更新中。

2025 年初，Anthropic 在推出 MCP（Model Context Protocol）之后，进一步提出了 Agent Skills 的概念。背后的思路很清楚：连接性（Connectivity）与能力（Capability）应该分离。

很多开发者认为”只要提示词写得好，AI 就能帮我做一切”。但事实是：Prompt 适合单次任务，Skills 才是构建可复用 AI 能力的正确做法。Skills 把 AI 应用从”个人技巧”拉到了”工程化”的层面。

正文

一、Skills 是什么

用一句话概括：Skill 是一个用自然语言定义的、具有特定领域上下文（Domain Context）的逻辑指令集，本质上是通过延迟加载（Lazy Loading）优化 Token 消耗的 Sub-Agent（子智能体）。

在团队协作中，很多”隐性知识”都在老员工脑子里，比如代码规范、排查流程、Review 标准。Skills 的核心价值，就是把这些隐性规则变成显性的文档（SOP），让 AI 能够自主阅读、理解并执行。

与传统编程不同，Skills 不强制规定每一步的代码逻辑，而是用自然语言将决策权下放给模型——模型通过读取 SKILL.md 后，将其中定义的规则、流程和约束实时注入到推理上下文中，指导后续的工具调用和决策。这既保留了 Agent 处理不确定性的优势，又避免了纯代码编排的僵化。

两个关键机制：

延迟加载（Lazy Loading）：元数据保持简短常驻上下文，正文仅在触发时动态注入，避免挤占 Token。

动态上下文注入：不同于静态文档的”阅读”，Skills 是将规则实时注入推理上下文，直接影响模型决策。

二、为什么需要 Skills

问题一：通用 AI 在垂直领域表现不稳定

即使是能力很强的大模型，在处理特定业务场景时也会表现不稳定。比如让 AI 做代码 Review，它可能今天关注安全问题，明天关注性能问题，后天又关注代码风格——缺乏一致性。

Skills 通过封装领域知识和执行 SOP，让 AI 在特定任务上的表现更稳定、更专业。

问题二：Prompt 难以复用和维护

每次对话都要重新写 Prompt，团队成员之间的 Prompt 质量参差不齐，好的 Prompt 也会随着对话上下文丢失。

Skills 提供了标准化的封装格式，可以跨项目、跨团队复用，像代码一样进行版本管理。

问题三：上下文窗口有限

如果把所有知识都塞进 System Prompt，很快就会撑爆上下文窗口，而且大量无关信息会干扰模型的判断。

Skills 通过渐进式披露（Progressive Disclosure）机制，只在需要时加载相关知识，大幅降低 Token 消耗。

三、Skills 长什么样

从结构上看，Skill 很简单，核心就是一个 SKILL.md 文件，包含元数据（描述什么时候用）和正文（具体的执行 SOP）。

四、Skills 与 Prompt、MCP、Function Calling 的区别

这是面试中常被问到的点，也是最容易混淆的地方。

Skills vs Prompt

Prompt 是用户即时表达意图的载体，随对话上下文丢失，难以维护。Skills 是包含元数据（何时使用）+ 正文（如何执行）的完整方案，可以持久化、跨项目复用，并通过延迟加载机制按需注入上下文。

Skills vs MCP

MCP 解决的是连通性问题：它像 USB-C，让 AI 能以统一格式读文件、查数据库、调用外部 API。Skills 解决的是编排逻辑问题：它像一份说明书，告诉 AI 如何执行复杂任务流——这些任务完全可以包括调用多个 MCP 工具。两者是互补关系，不是竞争关系。

Skills vs Function Calling

Function Calling 是底层机制，是原子操作（单次工具调用）。Skills 是上层应用，是复合流程（多步骤决策 + 工具组合）。Skills 在执行时可能会通过 Function Calling 来调用脚本、读取资源或调用 MCP 工具。

用一张表格总结四者的关系：

组件 一句话定义 形象类比 关键理解
Prompt 即时意图表达的载体 用户说的话 单次、易失
Function Calling LLM 输出结构化调用的能力 神经信号 一切的基础
MCP 标准化的工具接入协议 USB-C 接口 解决外部系统”如何接入”
Skills 用自然语言定义的 sub-agent 任务说明书 解决复杂任务”如何编排”

四层关系：Function Calling 是地基 → Prompt 表达意图 → MCP 负责连通外部系统 → Skills 负责编排复杂任务流（可调用 MCP）。

五、Skills 的典型应用场景

固化工程标准：将团队的代码规范、Review 标准、部署流程封装成 Skill，让 AI 严格执行，保持一致性。

领域专家知识：将特定业务领域的知识（如金融风控规则、医疗诊断流程）封装成 Skill，让通用 AI 具备专业能力。

复杂工作流自动化：将多步骤的复杂任务（如”分析日志 → 定位问题 → 生成修复方案 → 创建 PR”）封装成 Skill，实现端到端自动化。

知识库检索增强：将 RAG 检索流程封装成 Skill，让 AI 在回答问题前先检索相关文档，提高回答准确性。

一些优秀的开源 Skills 示例：

  • Code-Review-Expert：以资深工程师视角进行结构化代码审查
  • Git Commit with Conventional Commits：自动生成语义化 commit message
  • TDD：测试驱动开发流程

skills.sh 这个网站上可以查找和分享各种 Skills。

总结

Agent Skills 是 AI 能力扩展的新范式，它通过将领域知识和执行 SOP 封装成标准化的 SKILL.md 文件，解决了三个核心问题：通用 AI 在垂直领域表现不稳定、Prompt 难以复用和维护、上下文窗口有限。

Skills 的本质是延迟加载的 Sub-Agent：元数据常驻上下文让 AI 知道有哪些能力可用，正文按需加载避免 Token 浪费，脚本和资源文件提供确定性的执行保障。

与 MCP 的关系是互补而非竞争：MCP 负责连通外部系统，Skills 负责编排复杂任务流，两者配合使用才能构建真正强大的 AI Agent。
