第十三篇：什么是Agentic Workflow

发表于2026-04-06|更新于2026-04-29|大模型学习5-Workflow（工作流）
|浏览量:

前言

本文是「大模型学习」系列文章之一，持续更新中。

Agentic Workflow 是 AI Workflow 的进化形态，它将 Agent 的自主决策能力嵌入到工作流的关键节点，在保持整体流程可控的同时，赋予局部步骤更强的智能性。

什么是 Agentic Workflow

Agentic Workflow 是一种混合架构：

在 Agentic Workflow 中，工作流提供整体的执行框架和约束，而 Agent 节点在其负责的范围内自主决策，包括选择工具、决定执行次数、处理异常情况等。

Agentic Workflow 的核心特征

局部自主性：Agent 节点可以在其范围内自主决策，不需要预先定义所有可能的路径。

工具调用循环：Agent 节点内部可以多次调用工具，直到完成子任务。

动态适应：Agent 可以根据中间结果调整策略，而不是机械地执行预定步骤。

可观测性：整体工作流的执行路径是可追踪的，Agent 节点的内部决策也可以记录和审计。

Agentic Workflow 的典型模式

模式一：工作流中嵌入 Agent

模式二：多 Agent 协作工作流

Agentic Workflow 的设计原则

明确边界：清晰定义每个 Agent 节点的职责范围，避免职责重叠或遗漏。

最小权限：每个 Agent 只配备完成其子任务所需的工具，不要给予过多权限。

可中断性：在关键节点设置检查点，支持人工审核和干预（Human-in-the-Loop）。

错误处理：为 Agent 节点设置最大迭代次数和超时限制，防止无限循环。

可观测性：记录每个 Agent 节点的输入、输出和中间步骤，便于调试和审计。

总结

Agentic Workflow 代表了 AI 应用架构的演进方向：在保持整体流程可控的前提下，引入 Agent 的自主决策能力，处理那些无法预先定义所有步骤的复杂子任务。这种混合架构兼顾了可靠性和灵活性，是构建生产级 AI 应用的重要模式。
