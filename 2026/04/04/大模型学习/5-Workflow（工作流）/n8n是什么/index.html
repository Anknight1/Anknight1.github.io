第十一篇：n8n是什么

发表于2026-04-04|更新于2026-04-29|大模型学习5-Workflow（工作流）
|浏览量:

前言

本文是「大模型学习」系列文章之一，持续更新中。

n8n 是一个开源的工作流自动化工具，可以连接几乎任何 API 和服务，构建复杂的自动化流程。与 Dify 专注于 AI 应用不同，n8n 是一个通用的自动化平台，AI 能力是其众多能力之一。

n8n 的核心特点

开源自托管：n8n 完全开源，可以部署在自己的服务器上，数据不离开企业内网，满足数据安全要求。

400+ 集成：内置 400 多个服务的集成节点，包括 Slack、GitHub、Google Sheets、MySQL、PostgreSQL、Salesforce 等，几乎覆盖所有主流 SaaS 服务。

可视化编排：拖拽式界面，无需编写代码即可构建复杂的自动化流程。

代码节点：支持 JavaScript 和 Python 代码节点，处理复杂的业务逻辑。

AI 能力：内置 AI Agent 节点，支持 OpenAI、Anthropic、Google Gemini 等主流 LLM。

n8n 的核心概念

Workflow（工作流）：由多个节点组成的自动化流程，可以手动触发、定时触发或通过 Webhook 触发。

Node（节点）：工作流的基本单元，每个节点执行特定操作，如发送邮件、查询数据库、调用 API 等。

Trigger（触发器）：工作流的起点，定义何时启动工作流，如定时触发、Webhook 触发、邮件触发等。

Expression（表达式）：在节点配置中引用前一个节点的输出数据，格式为 {{ $json.fieldName }}。

n8n vs Dify 选择建议

选择 n8n 的场景：

  • 需要连接大量非 AI 服务（CRM、ERP、数据库等）
  • 工作流以数据处理和系统集成为主
  • 需要复杂的定时任务和事件触发
  • 团队有一定的技术背景

选择 Dify 的场景：

  • 主要构建 AI 对话和问答应用
  • 需要强大的 RAG 知识库功能
  • 团队中有非技术人员参与
  • 需要快速部署 AI 应用

总结

n8n 是一个强大的通用工作流自动化工具，其优势在于广泛的服务集成和灵活的自动化能力。对于需要将 AI 能力与大量外部系统集成的场景，n8n 是非常好的选择。
