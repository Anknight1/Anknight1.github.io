第二篇：上下文太长会带来什么问题

发表于2026-04-21|更新于2026-04-29|大模型学习6-上下文工程
|浏览量:

前言

本文是「大模型学习」系列文章之一，持续更新中。

现代大语言模型的上下文窗口越来越大：GPT-4 支持 128K tokens，Claude 3.5 支持 200K tokens，Gemini 1.5 Pro 甚至支持 100 万 tokens。窗口越大，是不是就越好？

答案是：不一定。上下文过长会带来一系列严重问题，这正是上下文工程要解决的核心挑战。

正文

一、Lost in the Middle：中间信息遗忘

2023 年，斯坦福大学的研究团队发表了论文《Lost in the Middle: How Language Models Use Long Contexts》，揭示了一个令人担忧的现象：

当关键信息位于长上下文的中间位置时，模型的性能会显著下降。

实验结果显示，模型对上下文的注意力呈现出明显的”U 型曲线”——对开头和结尾的内容记忆最好，对中间部分的内容记忆最差。

实验数据：

关键信息位置 模型准确率
上下文开头 ~71%
上下文中间 ~56%
上下文结尾 ~74%

这意味着：即使你把所有相关信息都塞进了上下文，如果关键内容恰好在中间，模型很可能”视而不见”。

根本原因：

Transformer 的注意力机制在处理极长序列时，注意力权重会被分散到大量无关 Token 上。关键信息（Signal）被海量的无关背景（Noise）稀释，导致模型无法有效聚焦。

二、上下文污染（Context Poisoning）

上下文污染是指错误、过时或矛盾的信息混入上下文，导致模型产生错误输出。

在 Agent 长时间运行的场景中，这个问题尤为突出：

常见的上下文污染场景：

  • 信息过时：早期对话中的事实在后续已经改变
  • 指令冲突：不同轮次的指令相互矛盾
  • 错误累积：Agent 的错误决策被保留在上下文中，影响后续判断
  • 噪声干扰：大量无关的工具调用结果占据上下文空间

三、上下文爆炸（Context Explosion）

在 AI Agent 场景中，每次工具调用都会向上下文追加新内容：

Manus 团队在实践中发现，Agent 的平均输入/输出 Token 比例约为 100:1——输入远远大于输出。这意味着上下文会随着 Agent 运行步骤的增加而急剧膨胀。

上下文爆炸的后果：

  • 超出模型上下文窗口限制，导致任务中断
  • API 调用成本急剧上升（按 Token 计费）
  • 响应延迟增加（处理更多 Token 需要更多时间）
  • 模型性能下降（过长的上下文反而干扰推理）

四、注意力稀释（Attention Dilution）

即使上下文没有超出窗口限制，过多的无关信息也会稀释模型对关键内容的注意力。

这是 Transformer 注意力机制的固有特性：

实际影响：

  • 模型遵循指令的能力下降（指令被”淹没”在大量内容中）
  • 事实回忆准确率降低
  • 推理链条容易断裂
  • 输出质量不稳定

Google 和 Percy Liang 的研究团队将这类现象统称为**”上下文退化综合症”（Context Degradation Syndrome）**。

五、成本与延迟问题

上下文长度与 API 成本、响应延迟成正比关系：

对于需要运行数百步的复杂 Agent 任务，不加控制的上下文增长会让成本变得不可接受。

六、上下文腐烂（Context Rot）

Manus 团队提出了”上下文腐烂”的概念：随着 Agent 运行时间增长，上下文中的有效信息密度逐渐降低，”腐烂”的无效信息越来越多。

上下文腐烂的表现：

上下文腐烂的危害：

  • 模型难以从大量历史信息中找到当前相关内容
  • Agent 开始”忘记”早期的重要指令
  • 任务执行出现前后矛盾
  • 模型可能陷入重复循环（因为”忘记”已经尝试过某个方案）

七、问题汇总与应对思路

问题类型 核心表现 应对策略
Lost in the Middle 中间信息被忽略 重要信息放首尾；使用结构化格式
上下文污染 错误/矛盾信息干扰 定期清理过时信息；版本化状态管理
上下文爆炸 超出窗口限制 摘要压缩；滑动窗口；外部存储
注意力稀释 关键信息被淹没 精简无关内容；提高信息密度
成本激增 API 费用不可控 Token 预算管理；按需加载
上下文腐烂 有效信息密度下降 定期重写上下文；结构化状态

这些问题共同指向一个核心需求：我们需要一套系统性的方法来管理 Agent 的上下文——这就是上下文工程（Context Engineering）要解决的问题，我们将在下一篇详细介绍。

总结

上下文过长会带来六类核心问题：Lost in the Middle（中间遗忘）、上下文污染、上下文爆炸、注意力稀释、成本激增和上下文腐烂。这些问题在 AI Agent 长时间运行的场景中尤为突出，是构建可靠 Agent 系统必须正视的挑战。

更大的上下文窗口不是万能药——如何精准地填充上下文，比拥有多大的上下文窗口更重要。
