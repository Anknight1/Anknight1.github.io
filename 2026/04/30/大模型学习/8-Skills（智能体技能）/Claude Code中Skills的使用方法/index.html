第三篇：Claude Code中Skills的使用方法

发表于2026-04-30|更新于2026-04-29|大模型学习8-Skills（智能体技能）

前言

本文是「大模型学习」系列文章之一，持续更新中。

前两篇介绍了 Agent Skills 的概念和渐进式披露机制。本篇进入实战，手把手讲解如何在 Claude Code 中安装、创建、使用和管理 Skills。

正文

一、Skills 的存放位置

Claude Code 支持三种 Skills 存放位置，适用于不同的使用场景：

位置 路径 适用范围 获取方式
个人 Skills ~/.claude/skills/<skill-name>/SKILL.md 所有项目 手动创建
项目 Skills .claude/skills/<skill-name>/SKILL.md 当前项目 手动创建
插件 Skills ~/.claude/plugins/cache/<marketplace>/<plugin>/skills/ 所有项目 插件市场安装

核心原则：无论哪种位置，每个 Skill 都要有自己的文件夹，不能把 SKILL.md 直接放在 skills/ 根目录下。

四、触发方式

Skills 支持两种触发方式：

自动触发（推荐）

Claude 会根据你的请求和 Skill 的 description 自动匹配。当你的请求与某个 Skill 的描述相符时，Claude 会自动加载该 Skill。

提高自动触发率的技巧：在 description 中包含用户可能说的关键词，描述要具体，明确说明使用场景。

手动触发

在提示中直接点名某个 Skill：

手动触发适合你明确知道要使用哪个 Skill，或者想降低自动匹配带来的歧义的场景。

注意：从 Claude Code 2.1 版本开始，Skills 和 Slash Commands 被统一合并，Skills 现在也可以通过 /skill-name 的方式手动调用。

插件安装后，Skills 会出现在 ~/.claude/plugins/cache/ 目录下，并被 Claude Code 自动识别。

七、安全注意事项

安装第三方 Skills 时需要注意安全风险，安装前务必检查：

检查 scripts/ 目录中的脚本内容，警惕以下危险信号：

  • 包含 curl 或 wget 下载外部文件
  • 包含 rm、delete 等删除操作
  • 尝试访问 ~/.ssh/、~/.aws/ 等敏感目录
  • 尝试上传数据到外部服务器

验证来源可信度：优先选择 stars 数多、有持续维护的仓库，以及 Anthropic 官方插件市场的 Skills。

  • Git Commit with Conventional Commits：自动生成语义化 commit message

文档处理类：

  • document-skills@anthropic-agent-skills：PDF、Word、Excel、PPT 处理

Claude Code 内置 Skills：

Skill 功能
/simplify 审查最近修改的文件，自动修复代码质量问题
/batch <指令> 大规模批量修改代码库，自动任务拆分
/debug [描述] 排查当前 Claude Code 会话问题

总结

在 Claude Code 中使用 Skills 非常简单：将 SKILL.md 文件放入对应目录，Claude 就会自动发现并在合适的时机加载它。

Skills 支持两种触发方式：自动触发（Claude 根据上下文匹配）和手动触发（直接点名 Skill）。安装来源包括官方插件市场、GitHub 仓库和手动创建三种方式。

最重要的是，Skills 的核心价值在于把重复的工作流变成可复用的资产——把团队的代码规范、部署流程、排查步骤封装成 Skill，让 AI 严格执行，保持一致性。
