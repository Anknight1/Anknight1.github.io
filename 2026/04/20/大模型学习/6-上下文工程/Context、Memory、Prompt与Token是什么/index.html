第一篇：Context、Memory、Prompt与Token是什么

发表于2026-04-20|更新于2026-04-29|大模型学习6-上下文工程
|浏览量:

前言

本文是「大模型学习」系列文章之一，持续更新中。

在构建 AI Agent 的过程中，你会频繁遇到几个核心概念：Context（上下文）、Memory（记忆）、Prompt（提示词） 和 Token（令牌）。它们既相互关联，又各有侧重。理解这四个概念的本质与边界，是掌握上下文工程的第一步。

正文

一、什么是 Token？

Token 是大语言模型处理文本的最小单位。模型不直接读取字符或单词，而是将文本切分为一个个 Token 再进行处理。

Token 的基本规律：

  • 英文中，1 个 Token 约等于 0.75 个单词（”hello world” ≈ 2 tokens）
  • 中文中，1-2 个汉字通常对应 1 个 Token（取决于具体的 tokenizer）
  • 标点符号、空格也会占用 Token
  • GPT-4 的上下文窗口为 128K tokens，Claude 3.5 支持 200K tokens

为什么 Token 很重要？

Token 数量直接决定了三件事：模型能处理多少信息、API 调用的费用、响应的延迟。每次调用 LLM API，输入和输出的 Token 总量不能超过模型的上下文窗口限制，超出则会报错或截断。

二、什么是 Prompt？

Prompt（提示词）是用户或系统向 LLM 发送的输入文本，用于引导模型产生期望的输出。

Prompt 通常分为三类：

类型 说明 示例
System Prompt 系统级指令，定义模型角色和行为规则 “你是一个专业的代码审查助手”
User Prompt 用户的具体请求或问题 “帮我审查这段 Python 代码”
Assistant Prompt 模型之前的回复（用于多轮对话） 模型上一轮的输出

Prompt Engineering（提示词工程） 是研究如何设计高质量 Prompt 的技术，包括 Few-shot 示例、Chain-of-Thought 推理链、角色扮演等技巧。

三、什么是 Context（上下文）？

Context 是一个比 Prompt 更宽泛的概念。Context 是 LLM 在推理时能够访问和使用的全部信息集合，它包含但不限于 Prompt。

一个核心类比：

如果把 LLM 比作 CPU，那么 Context Window 就是 RAM（工作内存）。CPU 只能处理当前在 RAM 中的数据，LLM 也只能”看到”当前 Context Window 中的内容。

这个类比非常贴切：

  • RAM 容量有限 → Context Window 有 Token 上限
  • 超出 RAM 需要换页到磁盘 → 超出 Context 需要外部存储（Memory）
  • RAM 中的数据是临时的 → Context 在会话结束后消失

四、什么是 Memory（记忆）？

Memory 是跨越 Context Window 限制的持久化信息存储机制。Context 是临时的（会话结束即消失），而 Memory 是持久的（可以跨会话保留）。

Memory 通常分为四种类型：

1. 短期记忆（Short-term Memory）

即当前 Context Window 中的对话历史，随会话结束而消失。这是最基础的”记忆”形式。

2. 长期记忆（Long-term Memory）

通过外部存储（数据库、向量数据库）持久化保存的信息，可以跨会话检索。典型实现：Mem0、Letta、LangGraph Memory。

3. 情节记忆（Episodic Memory）

记录具体事件和经历的记忆，例如”用户上次说他喜欢简洁的代码风格”。

4. 语义记忆（Semantic Memory）

存储通用知识和事实，例如知识库、FAQ 文档。通常通过 RAG 检索注入 Context。

五、四者的关系与边界

用一张表格来梳理四个概念的关系：

概念 本质 生命周期 存储位置 作用
Token 文本的最小处理单位 即时 模型内部 计量单位，决定容量和成本
Prompt 发送给模型的输入文本 单次请求 Context Window 引导模型行为和输出
Context 模型推理时的全部信息 会话级 Context Window（RAM） 决定模型”当前能看到什么”
Memory 跨会话的持久化信息 长期 外部存储（磁盘） 让 Agent 拥有”记忆”

Context 是 Memory 和 Prompt 的”交汇点”——Memory 中的信息需要被检索出来、注入到 Context 中，模型才能”看到”并使用它。这个”注入”的过程，正是上下文工程要解决的核心问题。

总结

  • Token 是文本的最小处理单位，是衡量 Context 容量和 API 成本的基本单位
  • Prompt 是发送给模型的输入文本，是 Context 的重要组成部分
  • Context 是模型推理时能访问的全部信息，受 Context Window（Token 上限）约束
  • Memory 是超越 Context Window 的持久化存储，需要通过检索机制注入 Context
