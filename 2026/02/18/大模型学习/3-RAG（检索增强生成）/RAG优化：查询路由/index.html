前言

本文是「大模型学习」系列文章之一，持续更新中。

实际的 RAG 系统往往有多个数据源（产品文档、FAQ、数据库、网络搜索），不同类型的问题应该路由到不同的数据源。查询路由（Query Routing）就是解决这个问题的技术。

什么是查询路由

总结

查询路由是构建多数据源 RAG 系统的关键技术：

  • 关键词路由：简单快速，适合规则明确的场景
  • LLM 路由：语义理解强，但增加延迟
  • Embedding 路由：速度和语义理解的平衡
  • 并行检索：不确定路由时，并行检索多个数据源

实际项目中，建议先用关键词路由快速实现，再根据效果逐步引入 LLM 路由。
