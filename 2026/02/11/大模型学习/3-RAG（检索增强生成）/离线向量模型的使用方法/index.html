前言

本文是「大模型学习」系列文章之一，持续更新中。

使用 OpenAI 等云端 Embedding API 存在几个问题：网络延迟、API 费用、数据隐私。本篇介绍如何在 Spring AI 中使用本地离线向量模型。

为什么需要离线向量模型

1. 数据隐私：企业内部文档不能发送到外部 API

2. 成本控制：大量文档向量化时，API 费用可观

3. 网络依赖：生产环境可能无法访问外网

4. 延迟优化：本地推理延迟更低（无网络往返）

常用的离线 Embedding 模型

模型 维度 语言 特点
BAAI/bge-m3 1024 多语言 支持中英文，效果好
BAAI/bge-large-zh-v1.5 1024 中文 中文效果最好
text2vec-large-chinese 1024 中文 国内常用
all-MiniLM-L6-v2 384 英文 轻量快速
e5-large-v2 1024 英文 效果好

总结

离线向量模型的三种使用方式：

  • ONNX 模型：直接在 JVM 中运行，推荐方案，无需额外服务
  • Ollama：本地模型服务，配置简单，支持多种模型
  • Python HTTP 服务：灵活，可使用任意 Python 模型

对于企业内部 RAG 系统，强烈推荐使用离线模型，既保护数据隐私，又降低成本。
