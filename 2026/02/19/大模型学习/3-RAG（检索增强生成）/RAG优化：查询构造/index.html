向量检索适合非结构化文本，但对于结构化数据（数据库、图数据库、API），需要将自然语言转换为对应的查询语言。查询构造（Query Construction）就是这个转换过程。

什么是查询构造

查询构造是将用户的自然语言问题转换为特定数据源的查询语言：

总结

查询构造将 RAG 的能力从非结构化文本扩展到结构化数据：

  • Text-to-SQL：查询关系型数据库，需要注意 SQL 注入安全
  • Text-to-Cypher：查询图数据库，适合关系型知识
  • Text-to-API：构造 API 调用参数，适合外部服务集成

关键是提供详细的 Schema 描述和示例，让 LLM 能够准确理解数据结构。
