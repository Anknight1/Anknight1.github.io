前言

本文是「大模型学习」系列文章之一，持续更新中。

本文手把手演示如何用 Spring AI 开发一个完整的 MCP Server + Client 系统，实现一个代码仓库智能助手：MCP Server 封装 Git 仓库相关操作（查询仓库信息、搜索代码、查看 PR），MCP Client 作为 AI 应用连接 Server，让用户用自然语言查询代码库。

整体架构

MCP Server 负责封装工具，MCP Client 负责连接 Server 并将工具注入 ChatClient，用户通过自然语言与 AI 交互，AI 自动调用合适的工具完成任务。

Part 1：开发 MCP Server

传输方式选择

Spring AI 提供了三种 MCP Server Starter，对应不同的传输方式：

Starter 传输方式 适用场景
spring-ai-starter-mcp-server stdio 本地进程，被 Claude Desktop 等直接启动
spring-ai-starter-mcp-server-webmvc SSE（Spring MVC） 独立 HTTP 服务，同步处理
spring-ai-starter-mcp-server-webflux SSE（WebFlux） 独立 HTTP 服务，响应式/异步处理

本文以 SSE（WebFlux） 方式为例，适合部署为独立服务。

总结

开发一套完整的 MCP Server + Client 系统只需几步：

Server 端：添加 spring-ai-starter-mcp-server-webflux 依赖，用 @Tool 注解定义工具，通过 MethodToolCallbackProvider 注册工具。

Client 端：添加 spring-ai-starter-mcp-client-webflux 依赖，在 application.yml 中配置 Server 地址，将 SyncMcpToolCallbackProvider 注入 ChatClient。

MCP 的核心价值在于标准化：Server 只需实现一次，就可以被 Claude Desktop、Cursor、自定义 Spring AI 应用等任何支持 MCP 协议的客户端使用，真正实现”一次开发，到处运行”。
