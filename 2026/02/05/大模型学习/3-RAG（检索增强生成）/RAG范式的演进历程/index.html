前言

本文是「大模型学习」系列文章之一，持续更新中。

RAG 技术自 2020 年提出以来，经历了三代范式演进。理解这个演进过程，有助于我们选择合适的 RAG 方案，并理解各种优化技术的来龙去脉。

第一代：Naive RAG（朴素 RAG）

基本流程

Naive RAG 的问题

检索质量问题：

  • 向量相似度不等于语义相关性
  • 用户问题表述不清时，检索效果差
  • 无法处理多跳推理（需要多个文档联合推理）

生成质量问题：

  • 检索到的文档可能包含矛盾信息
  • LLM 可能忽略检索内容，仍然产生幻觉
  • 无法处理检索结果为空的情况

Advanced RAG 在 Naive RAG 的基础上，在检索前和检索后增加了优化步骤。

检索后优化（Post-Retrieval）

重排序（Reranking）：

向量检索召回 Top-20，用 Cross-Encoder 重排序，取 Top-5。

上下文压缩（Context Compression）：

从检索到的长文档中，只提取与问题最相关的句子，减少噪声。

结果融合（Result Fusion）：

同时使用向量检索和关键词检索（BM25），融合两种结果（混合检索）。

如何选择

总结

RAG 的三代演进：

  • Naive RAG：简单直接，适合快速验证
  • Advanced RAG：在检索前后增加优化，显著提升效果
  • Modular RAG：模块化设计，灵活组合，适应复杂场景

后续文章将详细介绍 Advanced RAG 和 Modular RAG 中的各种优化技术。
