前言

本文是「大模型学习」系列文章之一，持续更新中。

向量检索（Bi-Encoder）速度快但精度有限，重排序（Reranking）使用更精确的 Cross-Encoder 模型对检索结果进行二次排序，显著提升最终结果的相关性。

Bi-Encoder vs Cross-Encoder

Bi-Encoder（双编码器）

向量检索使用的是 Bi-Encoder：

特点：

  • 文档向量可以预先计算并缓存
  • 检索速度极快（O(1) 查找）
  • 精度有限（问题和文档独立编码，无法捕捉细粒度交互）

Cross-Encoder（交叉编码器）

重排序使用 Cross-Encoder：

特点：

  • 问题和文档一起输入，能捕捉细粒度交互
  • 精度高（比 Bi-Encoder 高 10-20%）
  • 速度慢（无法预计算，每次都要重新计算）

总结

重排序是 RAG 系统中提升精度的关键技术：

  • 两阶段检索：Bi-Encoder 快速召回 + Cross-Encoder 精确重排
  • 效果显著：通常能将 Top-5 精度提升 10-20%
  • 代价：增加延迟（Cross-Encoder 计算量大）
  • 推荐方案：本地 BGE Reranker 或 Cohere Rerank API
