前言

本文是「大模型学习」系列文章之一，持续更新中。

传统 RAG 擅长检索独立的文本片段，但对于需要多步推理、理解实体关系的问题效果有限。Graph RAG 将知识图谱与 RAG 结合，显著提升复杂问题的回答能力。

Microsoft GraphRAG

GraphRAG 的两种查询模式

全局查询（Global Search）：

  • 适合宏观、总结性问题
  • 遍历整个知识图谱的社区摘要
  • 例如：”这个代码库的主要架构是什么？”

局部查询（Local Search）：

  • 适合具体、细节性问题
  • 从特定实体出发，遍历相关子图
  • 例如：”Spring AI 如何处理工具调用？”

Graph RAG vs 传统 RAG

对比维度 传统 RAG Graph RAG
多跳推理 弱 强
关系型问题 弱 强
构建成本 低 高（需要提取实体关系）
查询速度 快 较慢
适用场景 文档检索 知识密集型、关系型问题

总结

Graph RAG 通过引入知识图谱，解决了传统 RAG 在多跳推理和关系型问题上的局限：

  • 知识图谱：显式表示实体和关系，支持图遍历
  • 混合检索：图检索 + 向量检索，互补优势
  • Microsoft GraphRAG：目前最成熟的实现，支持全局和局部查询

适合知识密集型、需要多步推理的场景，但构建成本较高。
