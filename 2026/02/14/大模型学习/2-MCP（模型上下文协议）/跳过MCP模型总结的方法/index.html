前言

本文是「大模型学习」系列文章之一，持续更新中。

默认情况下，Spring AI 在工具调用完成后，会将工具结果发回给 LLM，让 LLM 生成一段自然语言总结。但有时候我们不需要这个总结，希望直接返回工具的原始结果。本篇介绍如何实现这个需求。

默认的工具调用流程

这个总结通常是有价值的，但在某些场景下会带来问题：

  1. 增加延迟：多一次 LLM 调用，增加响应时间
  2. 增加成本：多消耗 Token
  3. 信息失真：LLM 可能对工具结果进行”创意发挥”
  4. 格式破坏：工具返回的是结构化 JSON，LLM 总结后变成了自然语言

什么时候应该跳过模型总结

应该跳过（returnDirect=true）的场景：

  • 工具返回结构化数据，前端需要直接解析（JSON/XML）
  • 工具返回的是文件内容或二进制数据
  • 工具执行的是简单查询，结果已经足够清晰
  • 对延迟敏感的场景（实时性要求高）
  • 工具结果会被另一个工具继续处理（不需要人类可读格式）

不应该跳过（默认行为）的场景：

  • 工具返回的数据需要解释说明
  • 需要将多个工具的结果整合成连贯的回答
  • 用户期望得到自然语言回复
  • 工具结果包含错误，需要 LLM 友好地解释

总结

跳过 MCP 模型总结的核心是 returnDirect 参数：

  • @Tool(returnDirect = true)：注解方式，最简单
  • FunctionToolCallback.returnDirect(true)：编程方式
  • 自定义 ToolCallingManager：最灵活，可动态控制

合理使用 returnDirect 可以显著降低延迟和成本，但要注意它会改变用户体验——用户收到的是原始数据而非自然语言回复。
