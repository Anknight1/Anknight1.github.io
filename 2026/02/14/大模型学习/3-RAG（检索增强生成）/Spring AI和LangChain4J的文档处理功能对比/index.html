功能对比总结

功能 Spring AI LangChain4J
PDF 加载 ✅ 内置（PagePdf/Paragraph） ✅ 需要 Apache PDFBox
Word 加载 ✅ Tika ✅ Apache POI
HTML 加载 ✅ Tika ✅ Jsoup
分片策略 TokenTextSplitter recursive/sentence/paragraph
元数据增强 ✅ Keyword/Summary Enricher 需自定义
向量数据库 20+ 种 15+ 种
RAG Pipeline QuestionAnswerAdvisor EmbeddingStoreContentRetriever
流式输出 ✅ ✅

如何选择

  • Spring Boot 项目：优先选 Spring AI，与 Spring 生态无缝集成
  • 需要更多 RAG 控制：LangChain4J 的 API 更细粒度
  • 快速原型：两者都很简单，Spring AI 的 QuestionAnswerAdvisor 更简洁

总结

Spring AI 和 LangChain4J 都提供了完整的文档处理功能，主要差异在于：

  • Spring AI 更注重与 Spring 生态的集成，提供了更多开箱即用的功能（如元数据增强器）
  • LangChain4J 的 API 更细粒度，控制更灵活
