前言

本文是「大模型学习」系列文章之一，持续更新中。

一个 MCP Server 可能暴露几十个工具，但在特定场景下，我们只希望 LLM 使用其中的一部分。本篇介绍如何在 Spring AI 中实现 MCP 工具过滤。

为什么需要工具过滤

场景一：减少 Token 消耗

工具定义会占用大量 Token（每个工具的 name + description + schema 可能有几百 Token）。如果有 50 个工具，光工具定义就要消耗几千 Token。

场景二：避免 LLM 误用工具

工具太多会让 LLM 困惑，可能选择错误的工具。只提供当前场景需要的工具，可以提高准确率。

场景三：权限控制

不同用户只能使用不同的工具。比如普通用户只能查询，管理员才能修改。

场景四：A/B 测试

对不同用户群体开放不同的工具组合，测试效果。

工具过滤的最佳实践

1. 按场景分组，而不是按权限

2. 工具数量控制在 10 个以内

研究表明，工具超过 10 个后，LLM 的工具选择准确率会明显下降。

3. 记录工具使用情况

总结

MCP 工具过滤有多种实现方式：

  • ChatClient 层过滤：最灵活，适合动态场景
  • ToolCallbackProvider 过滤：在注册阶段过滤，适合静态配置
  • 注解分组：代码可读性好，适合工具较多的场景
  • 动态意图分析：最智能，但实现复杂
  • 名称前缀过滤：简单有效，需要规范命名

实际项目中，建议结合配置文件的白名单/黑名单 + 场景分组，既灵活又可维护。
