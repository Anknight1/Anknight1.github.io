第一篇：RAG是什么

发表于2026-02-01|更新于2026-04-29|大模型学习3-RAG（检索增强生成）
|浏览量:

前言

本文是「大模型学习」系列文章之一，持续更新中。

大语言模型（LLM）虽然强大，但有两个根本性的局限：知识截止日期和无法访问私有数据。RAG（检索增强生成）正是为了解决这两个问题而生的。

什么是 RAG

RAG，全称 Retrieval-Augmented Generation（检索增强生成），是一种将信息检索与文本生成相结合的 AI 技术框架。

简单来说，RAG 的核心思想是：在 LLM 生成回答之前，先从外部知识库中检索相关信息，然后将检索到的内容作为上下文提供给 LLM，辅助其生成更准确的回答。

为什么需要 RAG

LLM 的固有局限

1. 知识截止日期

LLM 的训练数据有截止日期，无法了解最新信息。比如 GPT-4 的训练数据截止到 2023 年，它不知道 2024 年发生的事情。

2. 无法访问私有数据

企业内部文档、产品手册、客户数据等私有信息，LLM 在训练时根本没有见过，自然无法回答相关问题。

3. 幻觉问题

当 LLM 不知道答案时，它可能会”编造”一个听起来合理但实际上错误的答案，这就是”幻觉”（Hallucination）。

RAG 如何解决这些问题

RAG 给 LLM 提供了”参考资料”，让它在有据可查的情况下回答问题，大幅减少幻觉。

RAG 的基本工作流程

RAG 分为两个阶段：索引阶段和检索生成阶段。

一个直观的例子

假设你要构建一个公司内部知识库问答系统：

索引阶段：

  1. 将公司所有产品手册、FAQ、内部文档上传
  2. 系统自动将文档切分成小段落
  3. 用 Embedding 模型将每段文字转换为向量
  4. 存入向量数据库（如 Milvus、Pinecone）

查询阶段：

  1. 员工问：”我们的产品支持哪些支付方式？”
  2. 系统将问题转为向量，在数据库中搜索最相似的段落
  3. 找到产品手册中关于支付方式的段落
  4. 将问题 + 相关段落一起发给 LLM
  5. LLM 基于这些资料生成准确回答

RAG vs 微调（Fine-tuning）

很多人会问：为什么不直接用微调来让 LLM 学习私有数据？

对比维度 RAG 微调
数据更新 实时更新，添加文档即可 需要重新训练
成本 低（只需向量化） 高（GPU 训练）
可解释性 高（可以看到引用来源） 低（黑盒）
适用场景 知识密集型问答 特定风格/格式学习
幻觉控制 好（有参考资料） 一般

结论： 对于知识库问答、文档检索等场景，RAG 是更好的选择；对于学习特定写作风格、专业领域语言习惯，微调更合适。

RAG 的典型应用场景

  • 企业知识库问答：员工查询内部文档、政策、流程
  • 客服机器人：基于产品手册回答用户问题
  • 法律/医疗咨询：基于专业文献提供参考意见
  • 代码助手：基于代码库和文档提供编程建议
  • 新闻/研究摘要：基于最新资讯生成摘要

总结

RAG 是解决 LLM 知识局限性的核心技术：

  • 本质：检索 + 生成的结合，给 LLM 提供”参考资料”
  • 解决的问题：知识截止日期、私有数据访问、幻觉问题
  • 两个阶段：离线索引（文档 → 向量库）+ 在线检索生成
  • 优势：低成本、可实时更新、高可解释性

后续文章将深入介绍 RAG 的每个技术细节，包括索引构建、检索算法、优化技术等。
