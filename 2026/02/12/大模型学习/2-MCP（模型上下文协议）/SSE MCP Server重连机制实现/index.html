前言

生产环境建议

1. 设置合理的超时时间

2. 在 Server 端配置心跳

心跳间隔建议 15-30 秒，小于负载均衡器的空闲超时时间（通常 60s）。

3. 使用 Streamable HTTP 替代 SSE

如果使用 Spring AI 最新版本，优先考虑 Streamable HTTP，它天然支持无状态重连。

4. 监控连接状态

总结

SSE MCP Server 的重连机制涉及多个层面：

  • 协议层：SSE 原生支持 retry 和 Last-Event-ID
  • Server 端：心跳保活 + 事件缓存重放
  • Client 端：自动重连 + 重新握手
  • 生产建议：优先使用 Streamable HTTP，避免 SSE 的状态管理复杂性
