第十一篇：相似度搜索有哪些常见算法

发表于2026-02-12|更新于2026-04-29|大模型学习3-RAG（检索增强生成）
|浏览量:

前言

本文是「大模型学习」系列文章之一，持续更新中。

向量数据库的核心是高效的相似度搜索。当数据库中有数百万甚至数十亿个向量时，如何快速找到最相似的 Top-K 个？本篇介绍常见的向量搜索算法。

相似度度量方式

在介绍搜索算法之前，先了解如何衡量两个向量的相似度。

余弦相似度（Cosine Similarity）

  • 值域：[-1, 1]，越接近 1 越相似
  • 不受向量长度影响，只关注方向
  • 最常用，适合文本语义相似度

欧氏距离（Euclidean Distance / L2）

  • 值域：[0, +∞)，越小越相似
  • 受向量长度影响
  • 适合图像特征等场景

点积（Dot Product / Inner Product）

  • 计算最快
  • 需要向量已归一化（否则受长度影响）
  • 归一化后等价于余弦相似度

暴力搜索（Brute Force）

最简单的方法：计算查询向量与数据库中所有向量的相似度，取 Top-K。

优点： 100% 精确，实现简单

缺点： 时间复杂度 O(n×d)，n 是向量数量，d 是维度。百万级数据时极慢。

适用场景： 数据量小（< 10万）或需要精确结果

IVF（倒排文件索引）

IVF（Inverted File Index）是一种基于聚类的近似搜索算法。

原理

索引阶段：

  1. 用 K-Means 将所有向量聚类为 N 个簇（通常 N = √数据量）
  2. 记录每个向量属于哪个簇

搜索阶段：

  1. 找到查询向量最近的 M 个簇（M << N）
  2. 只在这 M 个簇内做暴力搜索

参数说明

  • nlist：聚类数量，越大精度越高，但索引构建越慢
  • nprobe：搜索时检查的簇数，越大精度越高，但搜索越慢

优点： 内存占用小，适合大规模数据

缺点： 精度不如 HNSW，需要调参

HNSW（分层可导航小世界图）

HNSW（Hierarchical Navigable Small World）是目前最流行的 ANN 算法，兼顾速度和精度。

原理

HNSW 构建一个多层图结构：

搜索过程：

  1. 从最高层的入口点开始
  2. 在当前层贪心地向目标移动
  3. 到达局部最优后，下降到下一层继续搜索
  4. 在最底层找到 Top-K 结果

参数说明

  • M：每个节点的最大连接数，越大精度越高，内存越大
  • efConstruction：构建时的搜索宽度，越大质量越好，构建越慢
  • ef：搜索时的候选集大小，越大精度越高，搜索越慢

优点： 速度快、精度高、支持动态插入

缺点： 内存占用较大（需要存储图结构）

适用场景： 大多数生产场景的首选

LSH（局部敏感哈希）

LSH（Locality Sensitive Hashing）通过哈希函数将相似向量映射到同一个桶中。

原理

优点： 内存占用小，适合超大规模数据

缺点： 精度较低，需要多个哈希函数来提高召回率

适用场景： 超大规模（十亿级）、对精度要求不高

算法对比

算法 精度 速度 内存 适用规模
暴力搜索 100% 慢 低 < 10万
IVF 高 快 低 百万~亿级
HNSW 很高 很快 中 百万~千万级
LSH 中 快 低 亿级以上

精度 vs 速度的权衡

ANN 算法都是近似搜索，存在精度损失。通过调整参数可以在精度和速度之间权衡：

召回率（Recall@K）：在 Top-K 结果中，真正最相似的向量占比。

实际应用中，Recall@10 > 0.9 通常已经足够好。

如何选择索引类型

总结

向量相似度搜索算法的核心是在精度和速度之间寻找平衡：

  • 暴力搜索：精确但慢，适合小数据量
  • IVF：基于聚类，内存友好，适合大规模
  • HNSW：图结构，速度和精度最佳平衡，生产首选
  • LSH：哈希方法，适合超大规模

对于大多数 RAG 应用，HNSW 是最佳选择。
