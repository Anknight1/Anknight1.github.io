第六篇：MCP调试工具

发表于2026-02-06|更新于2026-04-29|大模型学习2-MCP（模型上下文协议）

前言

本文是「大模型学习」系列文章之一，持续更新中。

开发 MCP Server 时，如何快速验证工具是否正常工作？如何查看请求/响应的原始消息？本文介绍 MCP 官方调试工具 MCP Inspector，以及其他常用调试手段。

MCP Inspector 是什么？

MCP Inspector 是 Anthropic 官方提供的交互式调试工具，专为 MCP Server 开发者设计。它提供：

  • 可视化界面，无需写代码即可测试工具
  • 实时查看 JSON-RPC 请求/响应消息
  • 支持 stdio 和 SSE 两种传输方式
  • 可以测试 Resources、Tools、Prompts 三类能力

安装与启动

MCP Inspector 无需安装，直接用 npx 运行：

启动后，浏览器会自动打开 http://localhost:5173。

界面功能介绍

MCP Inspector 界面分为三个区域：

左侧：连接配置

  • Transport Type：选择 stdio 或 SSE
  • Command/URL：填写 Server 启动命令或 SSE 地址
  • Connect 按钮：建立连接

中间：功能测试

  • Resources：列出并读取 Server 提供的资源
  • Tools：列出工具，填写参数并执行
  • Prompts：列出并测试提示模板

右侧：消息日志

  • 实时显示所有 JSON-RPC 消息
  • 可以看到完整的请求和响应内容

连接 SSE 方式的 Spring AI MCP Server

  1. 启动你的 Spring AI MCP Server（端口 8080）
  2. 打开 MCP Inspector
  3. Transport Type 选择 SSE
  4. URL 填写：http://localhost:8080/mcp/sse
  5. 点击 Connect

连接成功后，左侧会显示 Server 信息，中间会列出所有可用工具。

测试工具调用

以天气查询工具为例：

  1. 点击中间的 Tools 标签
  2. 找到 getWeather 工具，点击展开
  3. 在参数输入框填写：{"city": "北京"}
  4. 点击 Execute

常见问题排查

问题：Inspector 连接失败

解决：确认 MCP Server 已启动，端口正确。

问题：工具调用返回错误

解决：检查参数名称和类型是否与工具定义一致。

问题：stdio Server 无法启动

解决：确认 java 命令在 PATH 中，或使用绝对路径。

总结

MCP Inspector 是开发 MCP Server 的必备工具：

  • 开发阶段：用 Inspector 快速验证工具是否正常工作
  • 调试阶段：查看原始 JSON-RPC 消息，定位问题
  • 测试阶段：批量测试各种参数组合

配合 Spring AI 的 DEBUG 日志，可以完整追踪 MCP 调用的全链路。
