前言

默认情况下，MCP Server 使用 HTTP 明文传输，存在数据被窃听的风险。本篇介绍如何将 MCP Server 升级为 HTTPS，以及客户端如何配置信任自签名证书。

为什么需要 HTTPS

MCP Server 传输的数据可能包含：

  • 用户的查询内容（可能含敏感信息）
  • 工具执行结果（可能含业务数据）
  • 认证 Token（API Key、JWT）

如果使用 HTTP 明文传输，这些数据在网络中是可见的，存在安全风险。

HTTPS 配置检查清单

部署 HTTPS MCP Server 前，检查以下项目：

  • 证书有效期（建议至少 90 天，使用 Let’s Encrypt 自动续期）
  • 证书域名与访问域名匹配
  • TLS 版本（禁用 TLS 1.0/1.1，只允许 TLS 1.2/1.3）
  • Nginx SSE 配置（proxy_buffering off）
  • 防火墙开放 443 端口
  • HTTP 重定向到 HTTPS
  • HSTS 头（Strict-Transport-Security）

常见问题

Q: SSE 连接在 HTTPS 下断开？

A: 检查 Nginx 配置中是否设置了 proxy_buffering off 和足够长的 proxy_read_timeout。

Q: 客户端报 “PKIX path building failed”？

A: 自签名证书未被客户端信任，需要将证书导入 JVM 信任库或配置自定义 SSLContext。

Q: 证书过期了怎么办？

A: 使用 Let’s Encrypt + certbot 自动续期，或者设置证书过期提醒。

总结

将 MCP Server 升级为 HTTPS 有三种方案：

  • Spring Boot 直接配置：简单，适合开发/测试
  • Nginx 反向代理：推荐生产环境，SSL 终止在 Nginx 层
  • Docker + Traefik：容器化部署的最佳选择，自动管理证书

无论哪种方案，都要注意 SSE 的特殊配置（禁用缓冲），以及客户端对自签名证书的信任配置。
